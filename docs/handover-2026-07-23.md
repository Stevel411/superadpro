## The one that would have cost real money

**AdvantageLife refunds were structurally impossible.**

`REFUND_SHARES` in `app/stripe_service.py` had no `al_lifetime` key. `calculate_refundable()` falls back to `.get(product_kind, 0.0)`, so every AL join charge was written with `refundable_cents = 0`. The member refund endpoint filters `refundable_cents > 0` — so it never found a row and returned `no_refundable_charge`.

Every single AdvantageLife refund request would have hard-failed, while `/refund-policy` — live, public, and linked from *inside Stripe Checkout* where the buyer reads it at the moment of payment — promised **"100% back. No questions, no partial figure, no deductions."**

The first member to ask for their $100 back inside 7 days would have been refused by the system, leaving a chargeback as their only route — against a payment processor registered to a dissolved company.

Fixed: `al_lifetime: 1.00`, on the refund policy's own reasoning — nobody earns a commission on a join, so the company holds the full $100 and can return the full $100. Verified $100.00 in → $100.00 refundable, with all other products unchanged. (`8e2cdd5e9`)

## Schema — a standing trap, now visible

`/api/blog/me` returned 500 for an admin with lifetime access. `MySite.jsx` **rendered that as a paywall** — it read `data?.is_pro` before checking whether the load succeeded, so any API error was indistinguishable from "you have not paid." You were asked to buy a tool you own; `/join` correctly told you you were already in and bounced you to the dashboard. A loop.

**Root cause:** `blogs.admin_suspended` existed in the model but not in Postgres.

**`SKIP_MIGRATIONS=true` means once a table exists, nothing ever adds a column to it again.** Any field added to a model after its table was created is simply absent, and the first query touching it raises `UndefinedColumn` → 500 on whatever endpoint reads it. A table-level check cannot see this.

`schema-check` now diffs model columns against the live database per table. Two columns were missing platform-wide; both added.

> **Run `/admin/api/schema-check` after any deploy that touches `database.py`.** Read-only by default, takes two seconds. This is the single highest-value habit to come out of today.

## Custom domains — v2 auto-TLS live

`custom_domains.py` was handing members `superadpro-production.up.railway.app` as their CNAME target — hardcoded, pointing at a service being decommissioned. Two more in the same file: `verify_cname` accepted only SuperAdPro's apex/www as alternates, so an AL member who sensibly CNAME'd to `advantagelife.club` was told their DNS was broken; and `is_blocked_domain` did not block AL's own domains, so a member could have claimed `www.advantagelife.club` against your own Railway service.

`RAILWAY_API_TOKEN` was missing, silently dropping the feature into **v1 DNS-fallback** — a mode that looks healthy from your side (cron reports success, CNAME verifies, row flips to "verified") while **no certificate is ever issued**, so the member's own branded site serves a browser security warning. Invisible to you, visible only to them.

Now confirmed: `mode: v2-railway-auto-tls`, `api_reachable: true`, `railway_domains: ["www.advantagelife.club"]` — which also proves the three `RAILWAY_*_ID` vars belong to AL, not SuperAdPro. A wrong service ID would not error; member domains would register against the wrong service and nothing would say so.

**Untested:** never run against a real member domain. Worth one end-to-end test on a domain you control before offering it.

## Traps worth remembering

- **`.get()` on Stripe SDK objects raises with the message `get`.** Use attribute access. Same family as `PaymentIntent.list()` throwing `KeyError: 0`. I hit this despite it being in my notes.
- **Check the enclosing shell before "fixing" a component.** A hardcoded green "Qualified" badge in `Watch.jsx` looked like a serious bug — it sits inside a `quotaComplete` early return and was correct. I reverted before shipping.
- **`.mk i` and similar class names exist across multiple server-rendered templates in `main.py`.** Assert match counts before replacing; a global replace reaches into pages you are not looking at.
- **Removing JSX can leave empty conditionals.** `{!data.has_access && ( )}` broke the build. The mandatory pre-push `npm run build` caught it; skipping that step ships a stale bundle silently.
- **Server-side redirects beat client-side ones.** Several retired URLs were loading the whole SPA then redirecting — two full page loads and a flash of empty layout.
