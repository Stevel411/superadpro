## The architectural fact that explains everything

AdvantageLife is **not a separate system**. It is the SuperAdPro codebase running from a branch against a fresh database:

- `app/main.py` — **73,666 lines**, of which **17** are gated by `IS_ADVANTAGELIFE`
- **88** React page components, **0** written for AL

Every SuperAdPro page came across by default. A page stops being live on AL only if someone deliberately removes, redirects or brand-gates it.

So the question is never *"is this page live?"* — it almost always is. It is **"has anyone dealt with this page yet?"**

## Traps worth remembering

- **`str_replace` refusing an edit as ambiguous is a feature.** `class="backlink" href="/dashboard"` exists in four templates sharing markup. Editing by exact line index with an assertion on the line content first was the safe path; a global replace would have broken three member pages.
- **A colour inversion is not finished until the leftover-token count reads zero.** The diff looked correct while six pieces of text were about to ship invisible.
- **Legacy octal escapes (`\2713`) are rejected in JS template literals.** The mandatory `npm run build` caught it; skipping that step ships a stale bundle silently and the page looks unchanged.
- **The FAQ is an accordion.** Answers are collapsed in the DOM until clicked, so a rendered-text scan reads questions only. Finding no answer content does not mean the answers are missing.
- **Route-exists ≠ page-reachable.** A static scan of `App.jsx` found 42 flagged routes; crawling the live site found 26 reachable pages, most of which bounce to `/` or `/login` when logged out. Crawl before scoping.
- **Comments count in a source grep.** The first residue scan flagged `/compensation-plan` for "Profit Grid" — from the comment explaining what the file replaced. Strip comments before drawing conclusions.
- **Verify `present_files` paths actually exist.** Two mockups were presented from a directory they had never been written to, and the links were dead.
